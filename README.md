# 🩺 ObGyn-LitRadar

מערכת אישית לאיסוף, קטלוג, דירוג וניתוח ביקורתי של ספרות מחקרית במיילדות וגינקולוגיה.

- **איסוף אוטומטי** מ-PubMed (חינמי, רשמי) מתוך רשימת עיתוני-על ועיתונים ייעודיים.
- **קטלוג** אוטומטי לנושאים (פוריות, אונקוגינקולוגיה, סיבוכי הריון, רצפת אגן ועוד) לפי MeSH.
- **דירוג חשיבות** שקוף (יוקרת עיתון × תכנון מחקר × טריות × ציטוטים × גודל מדגם).
- **ניתוח בלחיצת כפתור** — "המבקר הקליני": חוזקות, חולשות, הטיות, Spin, שורה תחתונה קלינית.

---

## התקנה (פעם אחת)

```powershell
cd "ObGyn-LitRadar"
.\run.ps1 -Setup            # יוצר סביבה וירטואלית ומתקין חבילות
copy .env.example .env      # ואז ערוך את .env והכנס ANTHROPIC_API_KEY
```

> לטקסט מלא דרך הפורטל המוסדי (שלב מתקדם) — ראה "חיבור הפורטל" למטה. לא חובה כדי להתחיל.

## הרצה

```powershell
.\run.ps1
```

הדפדפן ייפתח על האפליקציה. לחץ **"⬇️ משוך מאמרים חדשים מ-PubMed"** בסרגל הצד, ואז **"⚡ ניתוח מהיר"** על כל מאמר.

## 🗞️ הדייג'סט השבועי (אתר נבחר)

לצד אפליקציית Streamlit, יש **אתר דייג'סט סטטי, מעוצב ואינטראקטיבי** (`site/index.html`)
שמציג את **2-3 המאמרים החשובים ביותר מכל עיתון ליבה** (AJOG, Green Journal, BJOG,
NEJM, Lancet, ו-"White Journal"=IJGO). בנייה ופתיחה:

```powershell
.\run.ps1 -Digest          # בונה את site/index.html ופותח בדפדפן
# או ידנית:  python build_site.py  →  פתח את site/index.html
```

תכונות האתר: עברית RTL מלאה · חיפוש · סינון לפי נושא ולפי עיתון · מיון · תצוגה
**מדורגת או מקובצת לפי עיתון** · כרטיסי ניתוח נפתחים בלחיצה · צבע ייחודי לכל עיתון ·
תצוגת **הדפסה לג'רנל קלאב** (🖨) · רספונסיבי למובייל · נגישות (מקלדת + ARIA).

כוונון: `DIGEST_PER_JOURNAL` ו-`DIGEST_WINDOW_DAYS` ב-`app/config.py`; עיתוני הליבה
מסומנים `"primary": True`. הדייג'סט נבנה מחדש אוטומטית בכל ריצה שבועית.

## איך זה עובד (זרימה)

```
PubMed (esearch+efetch)  →  קטלוג נושאים + ציון חשיבות  →  SQLite
                                                              │
                    לחיצת "ניתוח" → שליפת טקסט (OA/פורטל) → Claude "המבקר הקליני" → כרטיס
```

## מבנה הקוד

| קובץ | תפקיד |
|------|-------|
| `app/config.py` | **כוונון:** עיתונים, נושאים, משקלי דירוג, מודל |
| `app/ingest_pubmed.py` | איסוף וקטלוג מ-PubMed |
| `app/scoring.py` | חישוב ציון חשיבות |
| `app/prompts.py` | הפרומפט "המבקר הקליני" |
| `app/critic.py` | מנוע הניתוח (Claude) + מטמון |
| `app/fulltext.py` | שליפת טקסט מלא (OA + פורטל מוסדי) |
| `app/db.py` | SQLite |
| `streamlit_app.py` | ממשק המשתמש |

הכל מודולרי: לכוונן עיתונים/נושאים/דירוג — ערוך רק את `config.py`.

## חיבור הפורטל המוסדי (טקסט מלא — שלב מתקדם)

כרגע המערכת שולפת טקסט מלא רק ממאמרי **Open Access** (דרך Europe PMC, בלי login).
כדי לשלוף גם ממאמרים בתשלום דרך המנוי שלך ("הספרייה הרפואית לצוותי הקהילה"):

1. בדוק ידנית: התחבר לפורטל → טאב **DOI/PMID** → הדבק DOI → ראה לאן זה מגיע ומה ה-URL הסופי.
2. עדכן ב-`app/fulltext.py` את `PORTAL_DOI_URL_TEMPLATE` לתבנית ה-URL (עם `{doi}`).
3. בהרצה הראשונה הדפדפן ייפתח גלוי — **התחבר ידנית פעם אחת**; הפרופיל נשמר ב-`.browser_profile/`
   והאוטומציה תשתמש בו שוב בלי סיסמאות.

> זה מדמה קריאה אנושית רגילה, on-demand, מאמר אחד בכל פעם — כדי להישאר בתוך תנאי השימוש.

## סינון "ערך גבוה" בלבד

כברירת מחדל המערכת מושכת **רק** מחקרים משני-פרקטיקה: RCT, מטא-אנליזות, סקירות
שיטתיות והנחיות (מזוהים גם לפי PublicationType וגם לפי הכותרת, כדי לתפוס מאמרים
טריים). זה מצמצם את הנפח דרמטית (~פי 10). לכוונון: `HIGH_VALUE_ONLY` ב-`app/config.py`,
או הצ'קבוקס "רק ערך גבוה" בממשק. סף חשיבות מינימלי: סליידר בסרגל הצד.

## אוטומציה (תזמון שבועי — מותקן)

משימה שבועית רשומה ב-Windows Task Scheduler בשם **`ObGyn-LitRadar-Weekly`**
(כל יום ראשון 07:00) שמריצה את `weekly_fetch.ps1` → מושכת מאמרי ערך-גבוה חדשים
ל-`litradar.db` וכותבת ל-`fetch_log.txt`.

```powershell
Get-ScheduledTask -TaskName "ObGyn-LitRadar-Weekly"   # בדיקת סטטוס
Start-ScheduledTask  -TaskName "ObGyn-LitRadar-Weekly" # הרצה ידנית
Unregister-ScheduledTask -TaskName "ObGyn-LitRadar-Weekly"  # הסרה
```
לשינוי היום/השעה — Task Scheduler (taskschd.msc) או רישום מחדש.
